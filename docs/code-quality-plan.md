## Tier 1 — Security Hotfixes

### 1.1 Fix missing owner_id guards on visibility/access setters

**File:** `src/db/models.js:226-240`

`setVisibility()` and `setAccess()` modify model visibility and access settings without checking that the current user owns the model. `deleteModel()` at line 207 shows the correct pattern with `.eq("owner_id", userId)`.

```prompt
In src/db/models.js, the setVisibility() and setAccess() functions at lines 226-240
do not filter by owner_id before updating. Add .eq("owner_id", userId) to both
queries, matching the pattern used in deleteModel() at line 207.
```

### 1.2 Fix missing user_id guard on deleteSweep

**File:** `src/db/models.js:505-512`

`deleteSweep()` has no ownership guard, so any authenticated user could delete any sweep record.

```prompt
In src/db/models.js, add a userId parameter to deleteSweep() and add
.eq("run_by", userId) before the delete operation.
```

### 1.3 Replace dangerouslySetInnerHTML in QR code rendering

**File:** `src/ui/execute/index.jsx:2652`

The QR code SVG is injected via `dangerouslySetInnerHTML`. Since the SVG is algorithmically generated by an internal function (not user-controlled), this is low-risk but bypasses React's XSS protection.

```prompt
In src/ui/execute/index.jsx, replace the dangerouslySetInnerHTML used for QR
code SVG rendering with a useRef approach. Render the QR SVG by setting
ref.current.innerHTML = qrSvg(token) instead of using dangerouslySetInnerHTML.
```

## Tier 2 — Quick Wins

### 2.1 Fix seed initialisation to use seeded PRNG

**File:** `src/ui/execute/index.jsx:922`

The simulation seed uses `Math.floor(Math.random() * 1e9)` which contradicts the project's seeded PRNG mandate (AGENTS.md §9.4).

```prompt
In src/ui/execute/index.jsx line 922, replace the useState initialiser:
Math.floor(Math.random() * 1e9) with mulberry32(Date.now())() .
Import mulberry32 from '../../engine/distributions.js' if available, or define it inline.
```

### 2.2 Deduplicate toTitleCase

**Files:** `src/ui/shared/tokens.js:37`, `src/ui/editors/index.jsx:8`

`toTitleCase` is defined and exported from `tokens.js` but also defined locally in `editors/index.jsx`.

```prompt
In src/ui/editors/index.jsx, remove the local toTitleCase function at line 8
and import it from '../shared/tokens.js' instead (it is already exported from there).
```

### 2.3 Fix unstable React key generation

**File:** `src/ui/editors/index.jsx:679`

`Date.now() + Math.random()` for React keys can produce duplicates at millisecond precision.

```prompt
In src/ui/editors/index.jsx line 679, replace:
  id: 'r' + Date.now() + Math.random()
with:
  id: 'r' + crypto.randomUUID()
```

### 2.4 Replace array index React keys (dynamic lists)

**Files (~28 locations):** `src/ui/execute/index.jsx`, `src/ui/editors/index.jsx`, `src/ui/editors/AiGeneratedModelPanel.jsx`, `src/ui/shared/components.jsx`, `src/ui/execute/BottomPanel.jsx`, `src/ui/execute/ExecuteCanvas.jsx`, `src/ui/execute/ExecuteActivityNode.jsx`, `src/ui/editors/ModelDiffPreview.jsx`

Array index keys (`key={i}`, `key={index}`) cause unnecessary re-renders and incorrect DOM reconciliation when arrays are filtered, sorted, or have items inserted/removed. Not all instances are problematic — only those in dynamic (sortable/filterable/mutable) lists.

```prompt
Audit all key={i} usage across src/ui/ for dynamic lists (filterable, sortable,
or mutable arrays). Replace with stable unique IDs from the data. Static lists
that never change order can safely keep key={i}.
```

### 2.5 Remove stale runtime schema detection

**File:** `src/db/models.js:303-308`

Runtime schema detection checks `data[0].warmup_period === undefined` on every `fetchRunHistory` call. This was a migration compat check and is no longer needed.

```prompt
In src/db/models.js, remove the runtime schema detection block around lines
303-308 that checks for warmup_period === undefined and logs a console.warn.
All production databases should be migrated by now.
```

**File:** `src/db/models.js` (lines 85, 90, 95, 117, 335)

### 3.3 Add React.memo to leaf/presentational components

**Files:** `src/ui/execute/`, `src/ui/shared/components.jsx`

Leaf/presentational components re-render on every parent update. Adding `React.memo` prevents unnecessary re-renders.

```prompt
Wrap the following presentational components with React.memo:
- CustomerToken (ExecuteCanvas.jsx)
- ServerBay (ExecuteCanvas.jsx)
- PhaseTag (execute/index.jsx)
- Tag (shared/components.jsx)
- KpiCard (ExecuteCanvas.jsx)
- MiniLineChart (BottomPanel.jsx)
- WaitHistogram (BottomPanel.jsx)
Ensure props are primitive or memoized to avoid breaking the memo comparison.
```

### 3.4 Refactor toRow/norm mapper with shared column map

**File:** `src/db/models.js:40-74`

The manual camelCase ↔ snake_case mapping in `norm()` and `toRow()` must be kept in sync manually.

```prompt
In src/db/models.js, define a shared COLUMN_MAP array at the top of the file
that lists all column mappings:
  { db: 'entity_types', js: 'entityTypes' }
Refactor norm() and toRow() to iterate over this map instead of
manually listing each field. This ensures they never drift out of sync.
```
